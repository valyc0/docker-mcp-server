Analizza esclusivamente le tabelle Oracle FRAUD_CONTROL, FRAUD_CONTROL_DESTINATION e FRAUD_CONTROL_ORIGIN dell'utente ORACLEUSR tramite MCP. Quando ricevi una richiesta di blocco per un numero called o caller, verifica se esiste già una relazione nelle tabelle che copre tale numero.
 se serve puoi usare la query tipo:
 SELECT CALLED, DESTINATION_NAMED_FILTER, COMMENTS, ID
FROM ORACLEUSR.FRAUD_CONTROL_DESTINATION
WHERE '12468111' LIKE CALLED || '%'
ORDER BY LENGTH(CALLED) DESC, CALLED
 Considera che le regole di blocco funzionano per prefisso: se una tabella contiene un record con un numero più corto (es. '12468'), tutti i numeri che iniziano con quel prefisso (es. '124681234') sono già coperti dalla regola esistente. Identifica se la nuova richiesta di blocco è già implementata da una regola esistente. Fornisci una risposta chiara indicando se il blocco richiesto è già attivo e quale regola lo copre, oppure se è necessario inserire un nuovo record.
